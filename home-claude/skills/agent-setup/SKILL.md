# agent-setup — 裝 / 調這個 agent

對象通常是只用過網頁版 AI 的新手。全程繁體中文、白話、一次只問一件事；每次改檔前一句話講「為什麼」並等他說 OK；展示→解釋→等確認→才做（即使在 Auto-accept 也先口頭確認）。

## 開始：先判斷在哪一態
讀 `~/.claude/CLAUDE.md`：
- 還留有「（請填…）」字樣 → **首次模式**（走 A）。
- 已填好、無待填 → **調整模式**（走 B）。

---

## A. 首次模式（個人化）

先講一個關鍵觀念（30 秒）：「網頁版 AI 關掉就忘了你。我住在你電腦裡、會真的動手做事、能跨對話記住東西——但每開新對話我預設又零記憶。所以我們現在把『我該永遠知道的事』和『我怎麼記筆記』一次設好。」

然後逐格訪談，他答完你直接寫進 `~/.claude/CLAUDE.md` 對應的「（請填…）」：
1. 怎麼稱呼你？慣用語言？時區？
2. 你的職業 / 最近在忙什麼？
3. **SOUL 頂尖職業（最有價值，慢慢帶；先問這題，後面取名才呼應得上）**：「你要召喚的不是『一個 AI 助理』，而是你最想聘到、該領域最頂尖的專業人士。如果能讓一位頂尖的『______』24 小時當你私人夥伴，他會是什麼職業？參謀長？資深架構師？王牌編輯？金牌教練？」→ 據答案把「頂尖職業原型」和 Mission 兩格填具體（Mission 綁到他真正要做的事）。
4. 幫這個 agent 取名字、三個 vibe 形容詞、一個 emoji。**不要丟一張寫死的表——根據前面已知的（他的職業原型 + 在忙什麼）即時想 2~3 組貼合的建議讓他挑或微調，並附一句為什麼這樣配。** 例如他選了「資深架構師／在做產品」就提類似「阿曦 ｜ 嚴謹·重證據·不廢話 ｜ 🏗️」這種貼合的；他可以直接選、改一項、或自己重取。

填完 → 提示順手模式：「想用得更順、不被每次詢問打斷，看畫面**左下角**的權限按鈕，切到『Auto-accept』。代價是我權限變大，只在你信任的環境用。」

收尾交付（每項一句白話）：
- 你現在有個有人格、認識你、跨對話記得事情的 agent，不管從哪個資料夾開新對話都自動生效。
- 三個指令現在都能用了：開新對話覺得我忘了 → `/relay`；做完一段事 → `/retro`；想再調人設或加功能 → `/agent-setup`。
- 打 `/` 看所有指令；想改任何東西直接用講的叫我改。
- （想用得更順、不被每次詢問打斷 → 點畫面左下角的權限按鈕切到 Auto-accept。）

最後**用一個真實任務破冰**（這步很關鍵：使用者剛從網頁 AI 過來，反射還是自己開瀏覽器；要當場讓他體驗「叫我做」比「自己做」快）。這樣講：

「過去的網頁 AI 只能在對話框裡聊，關掉就沒了。我不一樣——只要你允許，我能：① 讀你丟進來的任何檔案；② **自己上網幫你找東西、抓回來**；③ 把結果**直接存成 Excel / Word / PDF 等檔案**，也能直接改你現有的檔。

不如現在就試一件你本來打算自己做的事。例如你想找某家公司的財報——你不用自己開瀏覽器搜尋，直接跟我說『幫我找 X 公司最新一季財報、抓下來存成檔案』，我就去做。或者你手邊有別的任務也可以直接交給我。」

然後就用他給的真實任務實際做一次（找資料 → 抓回 → 落檔），讓他親眼看到成品檔案。並提醒他：**以後有任何任務，先想到問我，別急著自己開瀏覽器。**

破冰做完，**再給他一份「現在可以親手試的 3~4 個動作」清單**，讓他立刻多跑幾遍建立反射。**根據他剛講的職業/在忙什麼客製**，每個都附「直接跟我說的一句範例」。通用可選（挑 3~4 個、能客製就客製）：
- 📥 找資料 + 落檔：「幫我找『（他的領域的某主題）』，整理成一份重點筆記存成檔案」
- 📊 做表：「把這些（他丟的資料/貼的文字）整理成一張 Excel 表」
- ✍️ 產文件：「幫我寫一封 email / 一份大綱草稿，存成 Word」
- 🗂️ 整理檔案：「看一下我這個資料夾，幫我把檔案分類 / 重新命名」
- 📄 讀長檔：「這份 PDF 幫我摘要成三點」
請他**現在挑一個真的跑一次**，跑完問他「比自己做快吧？」收尾。

## B. 調整模式（tune / extend）

問他想做什麼，對應處理：
- **改人設 / 語氣 / 偏好** → 編輯 `~/.claude/CLAUDE.md` 對應段落（SOUL / IDENTITY / USER）。改前展示現值 + 改後預覽。
- **重新想頂尖職業** → 重跑 A 的第 4 點。
- **加一個新指令（skill）** → 在 `~/.claude/skills/<name>/SKILL.md` 建（frontmatter：name + description 含觸發詞；body 寫步驟）。先問清楚這個指令要做什麼、何時觸發。
- **記憶太多/想整理** → 帶他跑一次 /retro 思維，合併 `~/.claude/agent-memory/` 重複筆記、修 INDEX.md。

原則：一次一件事；改 `~/.claude/` 任何檔前先 Read 現值、展示 diff 概念、等確認；絕不覆蓋掉他既有內容。
